# 터널 라이닝 철근 자동화 워크플로우

Civil3D에서 도면을 전처리하여 JSON으로 추출한 뒤, Revit 플러그인에서 이를 불러와 3D 구조물과 철근을 자동으로 생성하는 전체 흐름을 정리한 문서입니다.

---

## Civil3D 전처리 (0:00 ~ 3:43)

### 1. 공통

#### 1-1. 좌표 이동
CAD 도면의 원점(0, 0)과 실제 구조도 위치 사이의 거리가 매우 크기 때문에, 블록 기준점을 원점 근처로 이동시켜 좌표 정밀도 손실을 방지합니다.

> 비고 : 사용자가 원하는 포인트를 기준점으로 자유롭게 지정할 수 있으며, 해당 점이 (0, 0, 0) 좌표가 됩니다.

#### 1-2. 블록 추출
Civil3D 도면 내 블록 참조를 탐색하여 터널 구조도가 포함된 블록들을 자동으로 식별하고 분해합니다. 각 블록의 이름과 위치 정보를 기반으로 시트 단위로 관리합니다.

#### 1-3. 사이클 바운더리
터널 사이클 영역을 정의합니다. 각 사이클의 좌우 경계선을 지정하여, 이후 철근 수집 및 구조물 추출 시 사이클별로 데이터를 분리할 수 있도록 합니다.

### 2. 터널 철근

#### 2-1. 철근 수집 및 정척 분할
사이클 바운더리 내에 포함된 철근 객체(Line, Arc, Polyline)를 자동 수집합니다. 연속된 세그먼트를 하나의 철근으로 병합하고, 분할하여 개별 철근 단위로 정리합니다.

#### 2-2. 중심선 객체 선택
터널 단면의 중심선을 사용자가 직접 선택합니다.

#### 2-3. 철근 텍스트 인식
도면 내 철근 규격 텍스트(예: H16, H19, H22 등)를 자동 인식하여 인접한 철근 객체와 매칭합니다. 이를 통해 각 철근의 정보를 자동으로 부여합니다.

### 3. 구조물 전처리

#### 3-1. 구조물 객체 선택
터널 라이닝의 구조물 단면을 구성하는 CAD 객체를 사용자가 선택합니다. 선택된 객체들의 좌표와 형상 정보를 추출하여 Revit에서 3D 솔리드로 변환할 수 있도록 준비합니다.

> 비고 1 : 테이블의 헤더 행(#, 면적, 사이클 등)을 클릭하면 오름차순/내림차순 정렬을 변경할 수 있습니다.
>
> 비고 2 : CAD 도면에서 선이 끊어져 있는 경우(휴먼 오류)가 발생할 수 있으며, 이러한 경우 경고 메시지 창을 통해 해당 사이클을 확인 가능합니다. (업데이트 예정)

### 4. 마무리

#### 4-1. JSON 추출
수집된 모든 데이터(구조물 형상, 철근 배치, 사이클 정보, 규격 텍스트 등)를 JSON 파일로 내보냅니다. 이 JSON 파일이 Civil3D와 Revit 사이의 데이터 교환 포맷으로 사용됩니다.

---

## Revit 후처리 (3:44 ~ 4:40)

### 1. 구조물 생성

#### 1-1. JSON 파일 선택
Civil3D에서 내보낸 JSON 파일을 Revit 플러그인에서 불러옵니다. 파일에 포함된 프로젝트명, 구간 정보, 사이클 목록 등을 파싱하여 UI에 표시합니다.

#### 1-2. 구조물 두께 설정
JSON에 저장된 2D 단면 형상을 이 두께만큼 Extrude하여 3D 솔리드 구조물을 생성하는 데 사용됩니다.

#### 1-3. 구조물 선택 후 생성 실행
JSON에 포함된 여러 구조물 영역 중 Revit에 생성할 항목을 선택하고 실행합니다. 선택된 구조물의 2D 단면을 Revit 좌표계로 변환한 뒤, 지정된 두께로 Extrude하여 Structural Framing 또는 DirectShape 요소로 생성합니다.

### 2. 횡방향 철근 배치

#### 2-1. JSON 파일 선택
Civil3D에서 내보낸 JSON 파일을 불러옵니다. 파일 내 `TransverseRebars` 항목을 파싱하여 철근의 좌표(Start/Mid/End), 세그먼트 타입(Line/Arc), 규격, 직경 정보를 수집합니다. 동시에 `StructureRegions`에서 구조도별 CTC 기본값을 자동으로 읽어옵니다.

#### 2-2. 구조도별 CTC 및 배치 철근 선택
각 구조도(구조도(1), 구조도(2), …)는 Expander 그룹으로 표시되며, 그룹 헤더에서 다음 정보를 확인·설정할 수 있습니다.

- 1Cycle / 2Cycle 철근 개수 및 합계
- **CTC (mm)** : 해당 구조도의 철근 간격. JSON에서 불러온 값을 기본값으로 사용하며 사용자가 직접 수정할 수 있습니다.
- 선택 개수 / 전체 개수
- 그룹 단위 전체 선택 / 해제 버튼

구조도 내부에서는 개별 철근(ID, 사이클, 규격, 직경, 세그먼트 수)을 체크박스로 선택·해제할 수 있습니다. 상단의 전체 선택 / 해제 버튼으로 모든 구조도의 철근을 한 번에 제어할 수도 있습니다.

> 비고 : Host 요소(Structural Framing)는 Comments 필드의 `구조도(N)` 패턴으로 자동 매칭되며, 돌출 깊이 역시 Host Comments에서 읽어옵니다.

#### 2-3. 배치 실행
선택된 철근을 구조도별 CTC 기준으로 Revit 내 Host 요소 위에 배치합니다. Line/Arc 세그먼트는 Revit의 Rebar API로 변환되며, 실패 시 Standard → FreeForm Rebar로 자동 폴백합니다. 배치 결과(성공/실패 개수, 직경별 통계, 구조도별 개수)는 완료 후 요약 창으로 출력됩니다.

### 3. 종방향 철근 배치

#### 3-1. JSON 파일 선택
JSON 파일을 불러옵니다. 프로젝트 정보, 구조도 개수, 내측·외측 곡선 수, 각 곡선의 호길이(mm)를 자동 분석하여 UI에 표시합니다. 내측 / 외측 분류는 Civil3D 원본 도면의 객체 생성 순서(앞 절반 = 내측, 뒤 절반 = 외측)를 기준으로 자동 수행됩니다.

#### 3-2. 구조도별 설정
각 구조도마다 다음 항목을 개별 설정합니다.

- **Pos1 (1번째 위치)** : CTC 간격을 측정할 **기준 곡선**을 선택합니다.
  - 내측 (Inner) : 내측 횡방향 철근 곡선 위에서 CTC 측정
  - 중앙 (Center) : 내측과 외측의 중점을 이은 중심 곡선 위에서 CTC 측정
  - 외측 (Outer) : 외측 횡방향 철근 곡선 위에서 CTC 측정
- **Pos2 (2번째 shift)** : 기준 곡선에서 부호 있는 shift를 추가로 적용합니다.
  - `+offset/2` : BC 반대 방향(지반 쪽)으로 이동
  - `0` : 기준 곡선 그대로
  - `-offset/2` : BC 쪽(터널 공간 쪽)으로 이동

  Pos1 × Pos2 조합은 **CTC 측정 기준선을 달리하기 위한 9가지 표현**이며, 실제 철근의 배치 위치는 이어지는 과정에서 내측 / 외측 횡철근 곡선과의 교차를 통해 결정됩니다.

- **CTC (mm)** : 철근 간 중심 간격 (기본 200mm).
- **갯수** : 해당 구조도의 총 철근 수. 짝수(가능하면 4N+2 형태) 권장.
- **직경** : D10 ~ D38 중 선택.
- **offset (mm)** : 횡방향 철근과 종방향 철근이 서로 간섭하지 않도록 하는 기본 거리로, 직경 선택 시 `(평균 횡방향 직경 + 종방향 직경)` 값으로 자동 계산됩니다. 사용자가 직접 수정한 경우 이후 직경 변경에도 그대로 유지됩니다.
- **상태** : 입력값 검증 결과가 배지 형태로 표시됩니다 (준비됨 / 검증 OK · N개 / 입력 오류 등).

> 비고 : 화면 우측의 가이드 다이어그램은 Pos1 × Pos2 9개 조합이 실제 평면상 어느 위치에 대응하는지 시각적으로 보여줍니다.

#### 3-3. 상태 새로고침
설정값을 바탕으로 실제 Revit 배치 전 시뮬레이션을 수행하여, 지정한 갯수가 기준 곡선 길이 안에 실제로 들어가는지 검증합니다. 공간이 부족하거나 CTC / 갯수 조합이 맞지 않으면 실제 배치 가능한 수량이 미리 표시되어 사용자가 설정을 조정할 수 있습니다.

#### 3-4. 배치 실행
각 구조도의 설정에 따라 종방향 철근을 배치합니다. 처리 순서는 다음과 같습니다.

1. Pos1에 따라 기준 곡선 (arc a) 결정 및 연결
2. Pos2 shift 만큼 arc(a)를 offset 하여 arc(b) 생성
3. arc(b) 중앙에서 양방향으로 CTC 등분하여 샘플 포인트 추출
4. 각 샘플 포인트의 법선 방향으로 가상 법선선 생성
5. 가상 법선선과 내측·외측 횡철근 곡선의 교차점 (F_inner, F_outer) 계산
6. 교차점을 기준으로 외측 원은 내측 방향, 내측 원은 외측 방향으로 각각 `offset/2` 만큼 이동한 최종 위치에 철근 배치
7. Host 요소의 깊이만큼 돌출하여 Revit Rebar 요소 생성

배치 완료 후 구조도별 생성 개수, 직경별 통계, 실패 원인 분석이 요약 창과 로그 파일로 출력됩니다.
